# Theming and Localization

The register supports site-level visual customization without fragmenting the product. A venue can run the system in its own visual identity and preferred mode, while each operator keeps their own language preference on login.

  <img src="./assets/theming-and-localization.gif" alt="Theming and localization demo">

## What Users Can Change

- 16 built-in visual themes shared across the register UI
- Light and dark mode for the register UI
- Languages are per operator, with each user returning to their own language choice on login

## What It Gives to a Site

- A venue can align the register with its own brand and operating style instead of inheriting one fixed default look
- Multiple clients can run the same product without ending up with identical register surfaces
- Shared theming preserves site consistency while operator-level language keeps the UI usable across mixed teams

## Why It Matters

This makes the product adaptable without forking it. The same register platform can be deployed across different venues while still feeling deliberate, specific, and owned by the people using it.
